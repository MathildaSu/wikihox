It has been an open free software project since October 2008, but the project began in 2000 and was initially developed by [http://www.logilab.fr/ Logilab] for internal uses such as intranet, bug tracker and forge applications. As of 2012, CubicWeb is being used in large-scale semantic web and linked open data applicationshttp://data.bnf.fr/semanticwebSimon, A., Wenz, R., Michel, V., & Di Mascio, A. (2013). Springer Berlin Heidelberg. and international corporations.http://lists.cubicweb.org/pipermail/cubicweb/2012-October/001262.html == Concepts == The framework is entirely driven by a data model. Once the data model is defined, one gets a functional web application and can further customize the views (by default it provides a set of default views for each type of data). A cube is a reusable component defining specific features. For example a cube [http://www.cubicweb.org/project/cubicweb-forge forge] allows one to create one's own forge and the forge cube reuses the cubes comment, file, email, etc. Interesting general purpose cubes include dbpedia and openlibrary. The framework has been translated to English, French, Spanish and German (April 2011). == Features == * Semantic web: supports [[Web Ontology Language|OWL]]/[[Resource Description Framework|RDF]] * Multi-source: supports RQL, [[SQL]], [[LDAP]], [[Subversion (software)|Subversion]] and [[Mercurial]] * [http://www.logilab.org/project/rql RQL]: Relationship query language to ease data query * Migration tool: fits into agile development * View selection principle: the engine selects the best view to fit the content to display according to the context * Cubes library: a wide range of cubes are available on the forge ==See also== * [[Sesame (framework)|Sesame]] * [[Jena (framework)|Jena]] * [[Mulgara (software)|Mulgara]] ==References== {{reflist}} ==External links== *{{official website|http://www.cubicweb.org/}} *[http://www.cubicweb.org/blog/1238 CubicWeb blog] *[http://www.oscon.com/oscon2010/public/schedule/detail/13867 OSCON 2010 presentation of CubicWeb] *[http://www.semweb.pro French Semantic Web Conference] *[https://pypi.python.org/pypi?:action=browse&c=537 CubicWeb components in the Python Package Index (PyPI)] *[http://eswc-conferences.org/sites/default/files/papers2013/simon.pdf paper at Extended Semantic Web Conference 2013] [[Category:Semantic Web]]
