/ {{Start date and age|2015|01|02|df=yes}} | latest release version = 15.7.7 - ''production series'' | latest release date = {{Start date and age|2015|08|05|df=yes}} | latest preview version = 16.1 - ''development series'' | latest preview date = {{Start date and age|2015|07|02|df=no}} | frequently updated = Yes | kernel type = [[Monolithic kernel]] | license = [[BSD licenses|2-clause BSD]][http://opnsense.org/about/legal-notices/ OPNsense 2-clause BSD license] | working state = Current | website = {{URL|opnsense.org}} }} '''OPNsense''' is an [[open-source software]] [[Firewall (computing)|firewall]] distribution based on [[FreeBSD]]. It is typically used for [[Stateful firewall|stateful]] perimeter firewalls, security [[computer appliance|appliances]], [[intrusion detection system]]s, [[router (computing)|routers]], [[Cloud computing|cloud routers]], [[wireless access point]]s, [[DHCP]] and [[Domain Name System|DNS]] servers, [[VPN]] endpoints, and [[unified threat management]]-machines. The OPNsense project is a [[Fork (software development)|fork]] of [[pfSense]]. The OPNsense distribution can be downloaded for free{{cite web|title=OPNsense is an open source, easy to use firewall and routing platform|url=http://sourceforge.net/projects/opnsense/|website=http://sourceforge.net|publisher=sourfeforge|accessdate=19 February 2015}} and installed on computer [[hard disk]]s, [[Solid-state drive|SSD]]s, [[flash memory|Compact Flash]] cards as well as on [[USB flash drive]]s or [[Live CD]]s. It runs on a number of [[embedded system]]s and generic low power{{cite web|title=Netboard A10|url=https://www.deciso.com/netboard-a10/#.VPMsvi5odQI|website=deciso.com|publisher=Deciso B.V.|accessdate=1 March 2015}}{{cite web|last1=Schellevis|first1=Jos|title=Under the Hood: AMD G-Series SOC Delivers the Horsepower for Next Generation Firewalls|url=http://community.amd.com/community/amd-blogs/amd-business/blog/2015/02/11/under-the-hood-amd-g-series-soc-delivers-the-horsepower-for-next-generation-firewalls|website=community.amd.com|publisher=Advanced Micro Devices, Inc.|accessdate=1 March 2015}} [[personal computer]]s. OPNsense can be configured and upgraded through a web-based interface. It is focused on code quality and easy development, implementing the [[Model–view–controller|MVC]] paradigm. The OPNsense [[FreeBSD Ports|ports]], [[source code]] and [[Build automation|build environment]] are freely accessible without licensing costs and without special clauses attached.{{cite web|last1=Cao|title=Deciso Launches OPNsense, a New Open Source Firewall Initiative|url=https://www.freebsdnews.com/2015/01/07/deciso-launches-opnsense-open-source-firewall-initiative/|website=freebsdnews.com|publisher=FreeBSD News - January 7, 2015|accessdate=2 March 2015}}{{cite web|title=OPNsense|url=https://github.com/opnsense|website=github.com|publisher=GitHub, Inc.|accessdate=19 February 2015}}{{cite web|last1=Fichtner|first1=Franco|title=Development Workflow in OPNsense|url=http://lastsummer.de/development-workflow-in-opnsense/|website=http://lastsummer.de|accessdate=21 February 2015}} ==Name== The name was derived from the [[suffix]] of its progenitor's name [[pfSense]]{{cite web|url=//blog.pfsense.org/?p=114|title=So what does pfSense stand for/mean, anyway?|first=Chris|last=Buechler|date=June 21, 2007|work=pfSense Digest}} and ''open'', and stands for: ''"Open (source) makes sense."''{{cite web|url=http://opnsense.org/about/about-opnsense/|title=About OPNsense|date=January 2, 2015|work=OPNsense website}} The name was derived from the fact that it aims to keep the source base of a distribution like OPNsense open and free to developers, [[entrepreneur]]s and non-technical users as well. ==History== The project started on 2 January 2015{{cite web|url=http://opnsense.org/opnsense-version-15-1-released//?p=97|title=OPNsense 15.1-RELEASED|first=|last=|date=January 2, 2015|work=OPNsense website}} as a fork of [[pfSense]], which started as a [[Fork (software development)|project fork]] from [[m0n0wall]] in 2004.{{cite web|url=//www.pfsense.org/index.php?option=com_content&task=view&id=68&Itemid=76|title=pfSense Open Source Firewall Distribution - History}} The project gives three reasons for forking:{{cite web|title=OPNsense:So why did we fork?|url=https://wiki.opnsense.org/index.php/OPNsense:So_why_did_we_fork%3F|website=opnsense.org|publisher=OPNsense|accessdate=21 February 2015}} * '''Technical''' – focus on clean codebase that can be used by developers and is maintainable. * '''Community''' – build a thriving community that gives and shares. * '''License''' – a simple 2-clause BSD license for the sources and the tools to [[Build automation|build]] it. The OPNsense{{cite web|title=End of the m0n0wall project. Team endorses OPNsense|url=http://www.reddit.com/r/opnsense|website=reddit.com|publisher=reddit inc.|accessdate=2 March 2015}} project has been supported by [[Deciso]], an [[Information and communications technology|ICT]] solution provider for network appliances and middleware software.{{cite web|title=Who we are - As sponsor|url=http://www.deciso.com/company/#.VOYFWpE_kxA|website=opnsense.org|publisher=Deciso B.V.|accessdate=19 February 2015}} On February 15, 2015 the founder of the m0n0wall project officially announced the end of the project and recommended OPNsense, which "aims to continue the open source spirit of m0n0wall while updating the technology to be ready for the future."[http://m0n0.ch/wall/end_announcement.php m0n0wall - End of Development announcement; m0n0.ch] Since May 8, 2015, the [[HardenedBSD]] project teamed up with OPNsense to offer HardenedBSD-backed versions of OPNSense.{{cite news|last1=Webb|first1=Shawn|title=HardenedBSD Teams Up With OPNSense|url=https://hardenedbsd.org/article/shawn-webb/2015-05-08/hardenedbsd-teams-opnsense|accessdate=13 May 2015|publisher=The HardenedBSD Project|date=8 May 2015}} The free HardenedBSD project offers security enhancements such as [[Address space layout randomization|ASLR]], [[PaX#PAGEEXEC|PAGEEXEC]], [[NX bit|NOEXEC]], [[Position-independent code|Position Independent Executable]] (PIE), and hardening of [[memory protection]] and [[ptrace]]. According to OPNsense developers, OPNsense will benefit from these security enhancements and other hardened FreeBSD code. The first OPNsense images on top of HardenedBSD were released on June 10, 2015.{{cite news|last1=Webb|first1=Shawn|title=First Experimental OPNSense Images With HardenedBSD|url=https://hardenedbsd.org/article/shawn-webb/2015-06-10/first-official-opnsense-images-hardenedbsd|accessdate=11 June 2015|publisher=The HardenedBSD Project.|date=June 10, 2015}} ==Releases== {| class="wikitable" style="float:right; text-align:center; margin-left:1em; margin-right:0" |- ! The version number for the future release is provisional; in time of the scheduled release, the future version number (blue) becomes the version number of the latest stable release (green) accordingly. OPNsense releases are also given [[alliteration|alliterative]] [[code name]]s, using an adjective and the name of an animal. The code names are in alphabetical order, allowing a quick determination of which release is newer. The releases are timed to be approximately bi-annual, resulting in a summer and a winter release per year. Upgrades between releases have to be done from one release to the next release in a gradual manner, simply by updating a production installation over the calendary release mark.
